Antigravity Browser Control을 사용하여 아래 테스트를 실행해줘.

아래의 테스트 시나리오의 순서에 따라 테스트를 진행해줘.
Live View Mode 로 실행해서 브라우저화면에 테스트 진행상황을 볼 수 있게 해줘.
최종 결과는  실행 로그(Console Log)와 최종 성공/실패 리포트를 보여줘.
시나리오 ID에 정의된 테스트 케이스를 순차적으로 수행하고, 화면 스크린을 찍어줘.
에러가 발생한 경우에는 디버깅이 가능하도록 에러 메세지를 상세하게 기록해줘.

# URL: localhost:5173
# 테스트 디바이스 : PC, Chrome

# 테스트 시나리오
  1. 시나리오 ID : 1
    1) 로그인
       - URL 접속
       - 부서명은 "원천엔젤스", 역할은 "회계", 사용자는 "ang001", 비밀번호는 "0000" 입력
       - 로그인 버튼 클릭 및 화면 로딩 대기
    2) 지출결의서 작성
       - "지출결의서 작성" 메뉴 클릭
       - "기본정보" 탭에서 다음 데이터 입력 후 다음 버튼 클릭
          청구유형 : "청구지출", 청구요청별칭 : "자동 테스트"
       - "지출내역" 탭에서 다음 데이터 입력 후 다음 버튼 클릭
          목/세목 : "사례비"/"지휘자" 또는 "행사비"/"연주행사"를 선택
          지출내역 : "임의 지출내역", 금액: 100000이하의 10000 단위 random 금액 입력
       - "파일첨부" 탭에서는 다음 데이터 입력 후 다음 버튼 클릭
          "파일첨부" 버튼을 클릭하여 최근항목에서 임의의 이미지 파일 등록
          별칭에는 "영수증" 으로 입력
       - "최종확인" 탭에서는 다음 데이터 입력 후 결재요청 버튼 클릭
         "추가의견" : "테스트 청구합니다"
         서명이 없을 경우에는 V 표시로 서명을 그린다.
    3) 청구목록 조회
       - "청구목록 조회" 메뉴 클릭
       - 목록에서 최상위 row에서 상세보기 아이콘 클릭
       - 오픈된 리포트에서 데이터 확인
         . 상단의 서명테이블에 "기안" 상태뱃지 확인
         . 지출내역에 입력했던 "관","항","목","세목","지출내역","금액"과 일치여부 확인
         . 화면 하단의 floating bar에서 "결재의견" 버튼 클릭하여 "기안" 내용 확인
  1. 시나리오 ID : 2
    1) 로그인
       - URL 접속
       - 부서명은 "음악부", 역할은 "부장", 사용자는 "mus002", 비밀번호는 "0000" 입력
       - 로그인 버튼 클릭 및 화면 로딩 대기
    2) 결재 처리
       - "내결재목록 조회" 메뉴 클릭
       - 시나리오 1에서 작성한 내용과 일치하는 데이터의 "결재" 버튼 클릭
       - 오픈된 리포트에서 데이터 확인
         . 시나리오 1에서 입력했던 "관","항","목","세목","지출내역","금액"과 일치여부 확인
         . 화면 하단의 floating bar에서 "승인" 버튼 클릭
         . 서명이 없을 경우에는 V 표시로 서명을 그리고, "결재의견"에는 "자동 서명" 입력후 "처리" 버튼 클릭
       - "내결재목록 조회" 목록에서 방금 처리했던 데이터가 없는지 확인한다.
